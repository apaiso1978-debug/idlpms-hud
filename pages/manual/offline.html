ภาพรวมระบบ Offline

E-OS ถูกออกแบบให้เป็น Progressive Web App (PWA) ที่สามารถทำงานได้แม้ในสภาวะที่ไม่มีการเชื่อมต่ออินเทอร์เน็ต เพื่อรองรับโรงเรียนในพื้นที่ห่างไกลที่อาจมีสัญญาณไม่เสถียร

Service Worker

จัดการ caching และ network requests โดยอัตโนมัติ รองรับหลาย caching strategies

IndexedDB Cache

เก็บข้อมูลบน Browser แบบ persistent รองรับการทำงานออฟไลน์อย่างสมบูรณ์

Background Sync

ซิงค์ข้อมูลอัตโนมัติเมื่อกลับมาออนไลน์ พร้อมระบบ conflict resolution

สถาปัตยกรรมระบบ

1 Caching Strategies

Cache-First STATIC ASSETS

ใช้กับ CSS, JS, Fonts, Images - โหลดจาก cache ก่อนเสมอ

Network-First API CALLS

ใช้กับ API endpoints - พยายามโหลดจาก network ก่อน fallback ไป cache

Stale-While-Revalidate HTML PAGES

ใช้กับหน้า HTML - ส่ง cache ทันที แล้วอัพเดทใน background

2 Sync Engine

Optimistic Updates

บันทึกลง IndexedDB ทันที แสดง "Saved" ให้ user เห็น แล้วค่อย sync เมื่อมีเน็ต

Conflict Resolution

ใช้ Last-Write-Wins เป็นหลัก หรือ Manual merge สำหรับข้อมูลสำคัญ

Retry Logic

Exponential backoff สูงสุด 5 ครั้ง พร้อม priority-based queue

ไฟล์ที่เกี่ยวข้อง

ไฟล์ หน้าที่ สถานะ
sw.js Service Worker หลัก - จัดการ caching และ offline Active
manifest.json PWA manifest - ข้อมูลแอพและ icons Active
src/services/CacheService.js IndexedDB wrapper สำหรับ data caching Active
src/services/SyncEngine.js Background sync และ conflict resolution Active
src/services/DataService.js Abstract data layer รองรับ local และ AppsScript Active
